WEEK 01

🤖 Welcome to Robotics! What IS a Robot?

Theme: Exploration & Discovery · Hardware Focus
🎯 Learning Objectives
  • Define what a robot is and give 3 real-world examples
  • Identify the main parts of the mBot2 robot (body, wheels, brain, eyes)
  • Understand input → process → output concept
  • Safely handle electronic components
🧠 Key Vocabulary
  • Robot — A machine that can sense, think, and act
  • Sensor — The robot's "eyes and ears"
  • Actuator — The robot's "muscles" (motors/wheels)
  • Controller — The robot's "brain" (the board)
📋 Pre-Class Checklist
✓
Charge all mBot2 units
✓
Install mBlock 5 on all laptops
✓
Print "Parts of a Robot" worksheet
✓
Prepare robot show-and-tell examples
📍 60-Minute Plan
🎉 Ice-Breaker: Robot Freeze Dance
Kids move like robots — stiff arms, beeping sounds. When music stops, freeze! Gets energy out and introduces "robot-like" movement.
⏱ 0–5 min
🎬 Video: Robots in Real Life
Watch a 3-min clip of robots (Spot the robot dog, Mars rover, robotic arm in a factory). Pause and ask: "What can YOU spot that the robot is doing?"
⏱ 5–15 min
🔧 Meet the mBot2! Unbox & Explore
Each child gets an mBot2. Guided exploration: name each part, touch the sensor, spin a wheel by hand. Draw their robot in engineer notebook.
⏱ 15–35 min
🃏 Parts Matching Card Game
In pairs, match picture cards of robot parts to their names and functions. First pair to match all 8 cards wins a "Robot Star" sticker.
⏱ 35–50 min
⭐ Reflection Circle
"Share one thing you discovered today." Students write their name on their mBot2 sticker label. Assign engineer notebooks.
⏱ 50–60 min
🏠 Take-Home Challenge
  • Find 3 robots at home or on TV. Draw them in your engineer notebook.
  • Ask a parent: "What do YOU think a robot is?"
🔧 Hardware Explorer
🎯 Robot Identifier
WEEK 02

🔩 Build It! Assembling Your Robot

Theme: Construction & Teamwork · Hardware Focus
🎯 Learning Objectives
  • Assemble the mBot2 chassis from components using step-by-step instructions
  • Attach wheels, motors, and the main board correctly
  • Practice patience, precision, and following instructions
  • Identify left vs. right motor orientation
🧠 Key Vocabulary
  • Chassis — The robot's frame/body structure
  • Motor — Converts electricity into movement
  • Screwdriver — Tool for fastening parts safely
  • Assembly — Putting parts together in order
📍 60-Minute Plan
🏗️ Warm-up: LEGO Speed Build
2-minute challenge — build the tallest tower using 10 LEGO bricks. Introduces the concept of "following a design" and working fast-but-carefully.
⏱ 0–8 min
📖 Demo: Instructor Builds First
Instructor assembles the chassis step-by-step on a document camera / big screen. Students call out each part name as it's added.
⏱ 8–18 min
🔧 Build Time! Let's Go!
Students assemble chassis in pairs using the illustrated guide. Instructor and assistant circulate. Celebrate each milestone with a "click" clap when a part snaps in.
⏱ 18–48 min
🔍 Quality Check & Troubleshoot
Partner checks: "Can the wheels spin freely? Is the board facing forward? Are all screws tight?" Fix any issues as a team.
⏱ 48–56 min
📸 Robot Portrait Day!
Each student holds their assembled robot for a photo (printed for their certificate later). Name their robot!
⏱ 56–60 min
🏠 Take-Home Challenge
  • Tell your family: "My robot is called _____ and it has ___ wheels."
  • Bonus: Find a picture of a real robot that looks like yours!
🔩 Master Assembler
🤝 Team Builder
WEEK 03

💻 First Code! Making It Move

Theme: Introduction to Programming · Coding Focus
🎯 Learning Objectives
  • Open mBlock 5 and connect the mBot2 via Bluetooth/USB
  • Use Move blocks to drive forward, backward, left, right
  • Understand that code = instructions for the robot
  • Run a program and observe the robot's reaction
🧠 Key Vocabulary
  • Code — Instructions written for a computer/robot
  • Block — A single instruction in Scratch/mBlock
  • Program — A set of blocks that runs in order
  • Debug — Finding and fixing errors in code
💡 Instructor Tips
  • Use the "Human Robot" game: whisper instructions to a child, they follow exactly — even silly ones. Shows why code must be precise!
  • Let kids discover bugs naturally. Say "interesting! Why did it do that?"
  • Pair faster learners with beginners as "Code Helpers"
📍 60-Minute Plan
🎮 Human Robot Game
One kid is the robot, one is the programmer. Programmer says: "Move forward 3 steps. Turn right. Move forward 2 steps." Robot must follow EXACTLY — even if it walks into a wall!
⏱ 0–10 min
💻 mBlock Setup & First Connection
Open mBlock 5, select mBot2, connect via USB. Instructor demo first. Students connect their own robots. Celebrate the first successful connection!
⏱ 10–22 min
▶️ My First 3 Blocks!
Drag: "Move forward 1 second" + "Turn left 0.5 sec" + "Move forward 1 second". Hit run. Watch robot move! Students modify the time values and observe changes.
⏱ 22–42 min
🏁 Square Challenge
Can you program your robot to drive in a SQUARE? Use Move + Turn blocks. Hardest part: getting exactly 90° turns. First to complete gets to be "Code Captain" next week!
⏱ 42–56 min
📓 Engineer Log
Write/draw in notebook: "Today my robot learned to ___." Save their square program with their name.
⏱ 56–60 min
💻 First Coder
🏁 Square Maker
WEEK 04

🔁 Loops & Lights — Repeating Actions

Theme: Loops, Sequence & LED Control · Coding Focus
🎯 Learning Objectives
  • Understand why loops save time (don't repeat blocks 100x!)
  • Use the "Repeat" block to loop movements
  • Control the mBot2's LED lights (color & blink)
  • Combine move + lights in one program
🧠 Key Vocabulary
  • Loop — Repeating a set of instructions multiple times
  • Repeat block — Runs code a set number of times
  • Forever block — Runs code non-stop until stopped
  • LED — Light Emitting Diode — the robot's lights
📍 60-Minute Plan
🔁 Loop-de-Loop Warm Up
Stand up. Clap 3 times. Sit down. Repeat 4 times! Now imagine writing that down. Loops make it shorter — only write the instructions ONCE.
⏱ 0–8 min
⭕ Repeat Block: Drive in a Circle
Use Repeat(8) + [Move forward 0.5s + Turn right 0.5s]. Robot makes an octagon or circle! Students experiment with repeat count to change the circle size.
⏱ 8–28 min
💡 Light Show! RGB LED Control
Add LED color blocks inside a Repeat loop. Make a disco robot! Set light to red → wait 0.5s → blue → wait 0.5s → green. Repeat forever. Students choose their own colors.
⏱ 28–46 min
🎉 Robot Disco Challenge
Combine movement + lights in a single program. Play music in class. Whose robot has the best "dance"? Students vote (can't vote for own!).
⏱ 46–56 min
🧩 Code Detective
Show broken code on screen (missing repeat, wrong order). Students spot and fix the bugs. First to raise hand with the correct answer wins a sticker.
⏱ 56–60 min
🔁 Loop Master
💡 Light Artist
WEEK 05

👁️ Robot Eyes — Ultrasonic Sensor

Theme: Sensing the World · Sensors & Electronics Focus
🎯 Learning Objectives
  • Understand how ultrasonic sensors work (like bat echolocation!)
  • Read sensor distance values in mBlock
  • Program the robot to stop before hitting a wall
  • Use "If...Then" logic based on sensor data
🧠 Key Vocabulary
  • Ultrasonic — Sound waves beyond human hearing
  • Distance — How far away an object is (in cm)
  • If...Then — Conditional logic: "IF something, THEN do this"
  • Threshold — The "trigger" distance (e.g. stop if < 15cm)
⚡ How the Sensor Works
  • Sends out an ultrasonic pulse (like shouting "echo!")
  • Pulse bounces off object and returns
  • Robot measures time taken → calculates distance
  • Bats use the same trick to fly in the dark!
📍 60-Minute Plan
🦇 Echolocation Game
One child is blindfolded (the bat). Others clap slowly when they're far, fast when they're close. Blindfolded child navigates by sound. Intro to ultrasonic sensing!
⏱ 0–8 min
📡 Read the Sensor Live
Use mBlock monitor mode to display live distance values. Students put hand in front of sensor — watch the number change! Record values at 5cm, 10cm, 20cm, 50cm in their notebook.
⏱ 8–22 min
🛑 Smart Braking: Stop Before the Wall
Code: "Move forward forever. IF distance < 15cm, THEN stop + beep + red light." Use masking tape to mark a "danger zone" on the floor. Robot must stop in the zone!
⏱ 22–44 min
🏎️ Obstacle Course Challenge
Set up cardboard boxes as obstacles. Program robot to navigate without crashing. Closest stop to the wall without touching = winner!
⏱ 44–57 min
📓 Log It
Draw the sensor in notebook. Write: "The sensor works by ___."
⏱ 57–60 min
👁️ Sensor Whiz
🛑 Safe Driver
WEEK 06

🖤 Follow the Line — IR Sensor

Theme: Line Following · Sensors & Coding Combined
🎯 Learning Objectives
  • Understand how infrared (IR) sensors detect black vs. white surfaces
  • Program a basic line-following algorithm
  • Tune the robot's speed for smooth line following
  • Understand that line following is used in REAL factory robots!
🧠 Key Vocabulary
  • Infrared (IR) — Light invisible to humans, detected by sensors
  • Reflect — Light bouncing back (white surface reflects more)
  • Absorb — Light not bouncing back (black surface absorbs)
  • Algorithm — A step-by-step problem-solving recipe
📍 60-Minute Plan
⚫ Light Reflection Experiment
Shine a flashlight on black paper vs. white paper. Which reflects more? Relate to IR sensor. Students predict: "If the sensor sees BLACK, the robot should turn ___."
⏱ 0–10 min
🖤 Draw the Track!
Students use black masking tape to create their own line track on a white poster board. A curve, a straight, maybe a gentle turn. The robot must follow THEIR design!
⏱ 10–22 min
💻 Code the Line Follower
Use mBlock's built-in line-following extension. "IF left sensor = black AND right sensor = white, THEN turn left." Discuss why this works. Students adjust speed.
⏱ 22–44 min
🏁 Line Race!
Each pair's robot follows their custom track. Timed! Best time wins "Speed Bot" trophy sticker. Discuss: why did some robots follow better? (tuning, track shape)
⏱ 44–57 min
🤯 Real World Link
Show photo of Amazon warehouse floor-following robots. "Your robot does EXACTLY what those $500,000 machines do!"
⏱ 57–60 min
🖤 Line Follower
⚡ Algorithm Designer
WEEK 07

🔊 Sound & Smarts — Making Robots React

Theme: Sound Sensor + If/Else Logic · All Skills Combined
🎯 Learning Objectives
  • Use the sound sensor to detect loud vs. quiet sounds
  • Apply if/else logic to make the robot react to claps
  • Combine sound, movement, and lights in one program
  • Plan and document a multi-sensor program in their notebook
🧠 Key Vocabulary
  • If/Else — Two paths: if condition true → do A, otherwise → do B
  • Decibel (dB) — Unit measuring how loud a sound is
  • Threshold — The "trigger" level for detecting a clap
  • Multi-sensor — Using more than one sensor together
📍 60-Minute Plan
👏 Clap-Controlled Warm Up
Instructor controls student "robots" with claps. 1 clap = move forward. 2 claps = turn. 3 claps = stop. Kids take turns being the robot and programmer.
⏱ 0–8 min
📊 Sound Level Meter
In mBlock monitor mode, whisper near sensor vs. shout. Read dB values. Find the threshold where a clap registers. Write that number in notebook.
⏱ 8–20 min
👏 Clap-to-Go Robot!
Code: "IF sound > 60dB THEN move forward 1s + green light. ELSE stop + red light." Test with claps from different distances. Students tune their threshold.
⏱ 20–42 min
🧠 Design Challenge: Clap Sequence
Advanced: 1 clap = go, 2 claps = turn, 3 claps = stop. Students plan this in their notebook FIRST, then code it. Instructor helps with nesting if/else.
⏱ 42–55 min
🌟 Share & Celebrate
Each student demos their clap-robot to the class. Preview: "Next week = FINAL SHOWCASE! Bring your ideas!"
⏱ 55–60 min
🔊 Sound Programmer
🧠 Logic Thinker

🏆 Olympiad Prep Challenges

Extra challenges for advanced students — or as Week 8 bonus events!

What is a Robotics Olympiad?

A Robotics Olympiad is a competition where student teams design, build, and program robots to complete specific challenges — navigating mazes, picking up objects, following lines, or reacting to their environment. Competitions happen at local, national, and international levels!

🏁 Challenge 1: Maze Navigator Easy
Set up a simple maze with cardboard walls. Program your robot to reach the end without touching any walls. Use your ultrasonic sensor to detect walls on the sides! Time limit: 2 minutes. Score: 10pts for completion, bonus 5pts for speed.
⭕ Challenge 2: Precision Parking Easy
Draw a parking space on the floor with tape (40cm x 25cm). Program your robot to drive forward and stop exactly inside the space. NO sensor allowed — pure timing code only! Closest to center wins. This tests precise programming skills.
🖤 Challenge 3: Line Dash Medium
A winding black tape track (approx 2 meters). Program your robot to complete it in the fastest time while staying on the line. Penalty: +2 seconds for each line crossing. Best time after 2 attempts wins. Tests sensor tuning and speed optimization.
👏 Challenge 4: Clap Command Relay Medium
Robot must travel from Point A to Point B using ONLY clap commands. 1 clap = forward, 2 claps = turn 90°, 3 claps = stop. Complete without touching the robot. Time starts on first clap, ends when robot crosses the finish line!
🤖 Challenge 5: Sumo Bot Hard
Two robots enter a circular arena (white circle, 1m diameter). Program your robot to push the opponent out of the ring using ultrasonic detection and evasion. Must stay inside the white boundary (IR sensor). 3 rounds, best of 3 wins. This is a REAL Olympiad event!
